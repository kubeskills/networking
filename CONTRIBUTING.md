# Contributing to k8s-networking-24h

Thank you for your interest in contributing!

## How to Contribute

1. **Report Issues:** Found a bug or error? Open an issue describing the problem
2. **Suggest Improvements:** Have ideas for better labs or explanations? We'd love to hear them
3. **Submit Pull Requests:** Fixed something or added content? Send a PR

## Areas Needing Help

- **Hour Content:** Many hours have template content that needs expansion
- **Manifests:** Add more comprehensive YAML examples
- **Scripts:** Improve automation and error handling
- **Diagrams:** Visual aids for complex concepts
- **Translations:** Non-English versions of content

## Development Setup

```bash
git clone https://github.com/YOUR_USERNAME/k8s-networking-24h.git
cd k8s-networking-24h
./setup.sh
```

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-lab`)
3. Test your changes thoroughly
4. Update documentation if needed
5. Submit PR with clear description

## Content Guidelines

- **Accuracy:** Test all commands and manifests
- **Clarity:** Use clear, concise language
- **Consistency:** Follow existing format and style
- **Completeness:** Include assessments and flashcards

## Code of Conduct

Be respectful, constructive, and welcoming to all contributors.
